1.3 데이터 추상화와 캡슐화

데이터 캡슐화(data encapsulation) 또는 정보 은닉(information hiding)은 외부 세계로부터 데이터 객체의 자세한 구현을 감추는 것.

데이터 추상화(data abstraction)는 데이터 객체의 명세(specification)와 구현(implementation)을 분리하는 것.

데이터 타입(data type)이란 객체(object)들과 이 객체들에 동작하는 연산(operation)의 집합.

추상 데이터 타입(ADT: abstract data type)은 객체의 명세와 이들 객체에 대한 연산의 명세가 객체의 표현과 연산의 구현으로부터 분리된 방식으로 구성된 데이터 타입이다.

예제 1.1 [추상 데이터 타입 NaturalNumber] 
ADT NaturalNumber is
객체: 
정수의 순서가 있는 부분 범위로서 
0부터 해당 컴퓨터의 최대 정수(MAXINT)까지 
값을 갖는다.
함수:
	모든 x, y는 NaturalNumber의 원소이고,
	TRUE, FALSE는 Boolean의 원소이고,
	+, -, <, ==, =는 일반적인 정수 연산에 대해

	Zero():NaturalNumber		::=	0
	IsZero(x):Boolean		::= 	if(x==0)IsZero=true
						else IsZero=false
	Add(x,y):NaturalNumber		::=	if(x+y<=MAXINT)Add=x+y
						else Add=MAXINT
	Equal(x,y):Boolean		::=	if(x==y)Equal=TRUE
						else Equal=FALSE
	Successor(x):NaturalNumber 	::=	if(x==MAXINT) Successor=x
						else Successor=x+1
	Subtract(x,y):NaturalNumber	::=	if(x<y)Subtract=0
						else Subtract=x-y
end NaturalNumber
